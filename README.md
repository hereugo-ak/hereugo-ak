```

     |
     +-- GFC '08 · Taper '13 · COVID '20 · '22 Hike Cycle

  6 BASE MODELS
     +-- TFT (Temporal Fusion Transformer)

     +-- 73.6% achieved accuracy
```

## Telegram Job Bot

<img src="https://img.shields.io/badge/Status-Coming_Soon-FF6B35?style=for-the-badge&labelColor=0f0f23&color=FF6B35" alt="status" />
<img src="https://img.shields.io/badge/Agents-Multi_Agent-9B59B6?style=for-the-badge&labelColor=0f0f23&color=9B59B6" alt="agents" />
<img src="https://img.shields.io/badge/Automation-Browser-58A6FF?style=for-the-badge&labelColor=0f0f23&color=58A6FF" alt="automation" />

</div>

> *Autonomous job application engine. Fetches, scores, and applies. No human in the loop.*

Multi-agent system that automates the full job application pipeline on Telegram.

| | |
|:---|:---|
| **Listing Aggregation** | Continuous ingestion from all major job platforms (LinkedIn, Naukri, Indeed, Glassdoor, AngelList). Normalized into a unified schema |
| **Scoring Engine** | Semantic matching against user CV using embedding similarity + LLM-based relevance scoring. Ranks listings by fit score |
| **Auto-Apply** | Browser automation submits applications using the user's CV and tailored responses. Headless Chromium with human-like interaction patterns |
| **Anti-Detection** | Fingerprint randomization, behavioral jitter, request throttling, residential proxy rotation. Designed to operate within platform rate limits |
| **Multi-Agent Architecture** | Scraper Agent (listings) -> Scorer Agent (ranking) -> Application Agent (submission) -> Monitor Agent (status tracking). Orchestrated via message queue |
| **Telegram Interface** | Real-time notifications for new matches, application status updates, and interview scheduling. Full control via chat commands |

<div align="center">

`Python` · `Playwright` · `LangChain` · `Telegram API` · `Chromium` · `Proxy Rotation` · `Embedding Similarity`

</div>
